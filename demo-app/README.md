## المسارين الرئيسيين

### 1. مسار العميل (Customer Path — HCE Mode)
يحوّل الهاتف إلى **بطاقة NFC** لاتلامسية:
- المستخدم يدخل المبلغ ورقم البطاقة
- SDK يُرمِّز البطاقة (Tokenization) ويُشفِّر المبلغ (AES-256-GCM)
- `AtheerApduService` تستعد للرد على أوامر APDU من جهاز القارئ
- المستخدم يُقرِّب هاتفه من جهاز نقطة البيع

### 2. مسار نقطة المبيعات (POS Path — SoftPOS Mode)
يحوّل الهاتف إلى **جهاز قارئ NFC**:
- المستخدم يدخل المبلغ المطلوب ويضغط "ابدأ القراءة"
- `AtheerNfcReader` يُفعِّل وضع قراءة NFC
- عند اقتراب بطاقة العميل أو هاتفه: يُرسَل SELECT AID ثم GET PAYMENT DATA
- يُستقبَل الرمز المميز المشفر ويُعرَض كنتيجة ناجحة

## تشغيل التطبيق

### من سطر الأوامر

```bash
# بناء التطبيق التجريبي (Debug APK)
./gradlew :demo-app:assembleDebug

# موقع ملف APK:
# demo-app/build/outputs/apk/debug/demo-app-debug.apk

# تثبيت مباشرة على جهاز متصل
./gradlew :demo-app:installDebug
```

## التقنيات المستخدمة

| التقنية | الاستخدام |
|---------|-----------|
| `Atheer SDK` | التشفير، الترميز، معالجة المعاملات، NFC |
| `Material Components` | واجهة مستخدم عصرية مع دعم RTL |
| `ViewBinding` | ربط Views بشكل آمن بدون `findViewById` |
| `Coroutines` | العمليات غير المتزامنة |
| `AES-256-GCM` | تشفير بيانات البطاقة عبر Android Keystore |
| `HCE (HostApduService)` | محاكاة بطاقة NFC |
| `NfcAdapter.ReaderCallback` | قراءة بطاقات NFC |
